# AI 驱动 vs 经验驱动：对比实验设计与结论

> 命题明确要求"比比看和靠经验拍脑袋有什么本质不同"。我们把它做成一个**受控实验**，而非口头论证。

## 1. 实验设计

- **同一输入**：相同品类（soundcore TWS 耳机）、相同 brief。
- **A 组（经验驱动）**：模拟传统 PM——单次"拍脑袋"，凭行业常识押注维度（音质/续航/性价比），不接数据管线、不做用户验证。见 `application/baseline/experience_driven.py`。
- **B 组（AI 原生）**：本系统全链路（JML→AMI→双路径概念→用户替身→行业专家→决策官）。
- **公平性处理**：A 组产出的概念也用**同一份真实数据**评估其"实际会达到的 NPS"（`predict_nps`），避免只比口号。

## 2. 量化维度

| 维度 | 含义 |
|---|---|
| 机会覆盖率 | 概念覆盖了多少 Top5 真实机会 |
| 命中真实痛点率 | 概念主打维度里有多少是数据验证的高机会痛点 |
| 证据引用数 | 结论可追溯到的真实证据条数 |
| 已验证假设数 | 经合成用户验证的假设次数 |
| 可行性风险识别 | 主动识别的技术/成本/合规风险数 |
| 预测 NPS | 数据驱动的 NPS 预测 |
| 耗时 | 跑通一轮所需时间 |

## 3. 样本数据下的结论（详见 `generated/运行报告.md` 第 8 节）

| 维度 | A 经验驱动 | B AI 原生 | Δ |
|---|---|---|---|
| 机会覆盖率 | 20% | 60% | +0.40 |
| 命中真实痛点率 | 33% | 100% | +0.67 |
| 证据引用数 | 0 | 56 | +56 |
| 已验证假设数 | 0 | 4 | +4 |
| 合成用户数 | 0 | 4 | +4 |
| 可行性风险识别 | 0 | 3 | +3 |
| 预测 NPS | -29 | +1 | +30 |

## 4. "本质不同"的五点

1. **从主观到可溯源**：每条结论带 `evidence_id` 并逐字命中校验，而非"我觉得"。
2. **从少量到全量**：统计真实评论全景，不被"最响的声音"带偏。
3. **从单次到可迭代**：工作流闭环，未达 GO 自动回炉修订。
4. **从拍脑袋到可量化**：ODI/Impact/NPS 把"重要性/未满足/价值"变成数字。
5. **从经验到系统能力**：方法论可沉淀为 SOP，复制给任意品类与任意人。

> 注意：合成用户与离线评分是"发现与排序"工具，不替代真实用户终验——这与 NN/g 的结论一致。AI 原生方法的价值在于"前置问题空间、把研究预算花在刀刃上"。

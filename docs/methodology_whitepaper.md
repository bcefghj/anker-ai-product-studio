# AI 原生产品定义方法论白皮书

> 把"产品经理拍脑袋"重构为"数据可溯源、可验证、可量化、可迭代"的工作流。
> 本白皮书描述方法论 SOP，并与传统经验驱动方法逐项对比。

## 0. 一句话

> **产品经理的天花板 = 产品判断的天花板。** AI 原生方法把"判断"建立在可溯源的真实证据、可验证的合成用户、可量化的机会评分之上，让天花板由"个人经验"升级为"系统能力"。

## 1. 方法论全景：映射安克三大平台

| 安克现有平台 | 传统形态 | 本系统的 AI 原生版 | 对应 Agent/模块 |
|---|---|---|---|
| **JML**（用户调研→需求） | 人工访谈、问卷、Shulex VOC | 真实评论 ABSA + ODI 机会评分 + 机会解决方案树 | `platforms/jml.py` |
| **AMI**（市场洞察） | 分析师做竞品/趋势报告 | 竞品 ABSA → 短板/白空间 + 趋势检索(Agentic RAG) | 超级智囊 `agents/super_think_tank.py` |
| **BEES**（体验评估） | 用户测试、NPS 调研 | 合成用户假设盲访谈 + 体验演化 + NPS 预测 | 用户替身 + `platforms/bees.py` |
| 第一性原理 | 资深判断 | 技术原生路径（端侧 AI / Thus 芯片能力正推） | 产品经理 `agents/product_manager.py` |

## 2. 五个 AI 角色（命题三角色 + 编排/治理）

1. **超级智囊（AMI）**：竞品逐家拆解、白空间识别、趋势检索。回答"机会在哪"。
2. **产品经理（编排）**：双路径概念生成 + 吸收反馈迭代收敛。回答"做什么"。
3. **用户替身（合成用户）**：OCEAN 画像、真实评论 grounding、**假设盲**反谄媚访谈。回答"用户认不认"。
4. **行业专家**：技术/BOM/供应链/合规可行性 + Reflexion 批判。回答"做不做得出来"。
5. **决策官**：NPS 预测 + GO/NO-GO + 可审计记录。回答"要不要做"。

## 3. 标准作业流程（SOP）

```
① 接入多源真实证据(Evidence) — 目标品牌 + 竞品评论 + 趋势
② JML 用户洞察: ABSA(确定性) → ODI 机会评分 → 机会解决方案树(OST)
③ AMI 竞品/趋势: 竞品短板 → 白空间机会; Agentic RAG 补证据
④ 概念生成(双路径): VOC 驱动 ⊕ 第一性原理/技术原生 → 融合概念
⑤ 用户替身验证: 假设盲访谈 → 接受度/异议/必须改进
⑥ 行业专家可行性 + Reflexion → 风险与缓解
⑦ 决策官: NPS 预测 → GO/NO-GO（HITL 闸口）→ DecisionRecord
⑧ Working Backwards 产出 PR/FAQ；评测 Rubric + AI/经验对比
   ↺ 未达 GO 且有迭代预算 → 回到 ④
```

## 4. 方法论锚点（均为业界成熟方法，非自造）

- **Working Backwards（Amazon）**：先写"上市新闻稿 + 内外部 FAQ"，写不动 = 概念不成立。
- **Opportunity Solution Tree（Teresa Torres）**：结果 → 机会 → 方案 → 实验，保证从洞察到方案可追溯。
- **ODI（Anthony Ulwick）**：`机会分 = 重要性 + max(重要性 − 满意度, 0)`，量化"未满足"。
- **VOC Impact**：`Impact = Reach × (Severity + Value + Strategic)`。
- **合成用户**：OCEAN 人格 + 社会心理 grounding + 假设盲反谄媚（SCOPE / Grounded Simulation / Synthetic Users）。

## 5. AI 驱动 vs 经验驱动（本质不同）

| 维度 | 经验驱动（传统） | AI 原生（本系统） |
|---|---|---|
| 洞察来源 | 个人经验、最响的用户声音 | 全量真实评论统计 + 竞品全景 |
| 可溯源性 | 弱，难追溯 | **每条结论带 `evidence_id`，逐字命中校验** |
| 用户验证 | 成本高、轮次少 | 合成用户面板，秒级多人格、假设盲 |
| 机会优先级 | 主观排序 | ODI/Impact 量化排序 |
| 速度 | 周级 | 秒级跑通一轮 |
| 可迭代 | 依赖人 | 工作流闭环，自动回迭代 |
| 确定性 | 因人而异 | 确定性核心 + 引用门槛，可控可审计 |
| 盲区 | "拍脑袋"押常识维度 | 数据暴露真实高机会痛点 |

> 量化证据见 `docs/generated/运行报告.md` 第 8 节与 `docs/ai_vs_experience.md`。

## 6. 落地与可推广

- **离线可复现**：无网络即可端到端跑，结果可审计；接入 MiniMax / 真实 Amazon 数据即增强。
- **品类无关**：换充电（Anker）/ eufy 仅需更换数据配置与 aspect 词典。
- **可沉淀为 SOP**：整套流程即一份可执行的"AI 原生产品定义 SOP"，可对接安克 AIME 平台作为新 Agent 能力。
